Start The Application


  • Usual start through the application launcher, you can launch with creating a new document or opening an existing document with suffix "msim"

  • When seeing the main window,you must select a sequence firstly : Accelerator->Sequences,if there is nothing, you should load an accelerator file : Accelerator->Load Default Accelerator/Load Accelerator,then select sequence.

Model Selection


  • There are three options and two sub-options under the Model menu :
    1. Design : Use the design values from the lattice files
    2. Live with RF Design : Use the values from the live machine except RF
    3. All Live : Use the values from the live machine
    • Magnet-Setpoint : Use the set values of the live magnet field
    • Magnet-Readback : Use the read-back values of the live magnet field

  • Note : When using Model 2 and Model 3, you must make sure that the channels are connected

Scan Feature


  • This feature allow you to vary parameters between two values with specified steps and generate several results automatically over the range of parameter values.
  • The generated records are assigned with names like : Scan M:[N1.N2], which means Scan number M, step N1 for parameter 1 and step N2 for parameter 2.
  • The upper limit of scannig steps is 100 to avoid the exceptions caused by superabundant running.

Table Display


  • There are five tables in the main view :
    1. History Records Table : the far left one, show all the history records and set record name
    2. New Run Parameters : show all the elements of the selected sequence with design values,live values if detect live channels,test values which set manually and scan settings.
      • note : scan values are always in the first priority and test value is in the second priority , no matter which model is selected.
    3. Parameter History : record the data used in simulation.
    4. Diagnostics History : there are two sub-tables, the left one show the live values of the selected diagnostic device, the right one show the history record of the diagnostic data.
    5. States Table : show the simulation results,filter by the control boxes.Since there are too many parameters to display,so we have a restriction to display at most two records in the table.
  • Rules :
    1. Can't show two history records which belong to different sequence in the table.
    2. The history records of the currently selected sequence are in the first priority to display.
    3. The latest history record is in the second priority to display.

Plot Instruction


  • Check boxes : show the graphs of the checked parameters automaticly
  • Show Difference : show the difference of two records when this button is checked.
  • Display Rules : same as table,see above,and at most display two records

Persistance


  • You can save the data in a document with suffix "msim", the data include model configuration, probe,results(records).
